General Usage

To add a dynamic form to your Wicket application simply create an object of type FormModel and configure it by adding input fields, validators, hint texts and so on in pure Java. Just follow these steps:

Create your FormModel object:

You define the structure and contents of your form dynamically at runtime by creating an object of type FormModel. Follow the links in the navigation menu for examples. Each example provides a "Show Java Code" Button that diplays the source code. The following example creates a form with a section containing a single text field:

FormModel myForm = new FormModel("Form Title");
myForm.add(new SectionModel("Section Title")
  .add(new TextFieldModel<String>("Text Field Label", String.class)));

Add the Form to the HTML Markup:

To include the form in you page, add a div (or any other container tag) to your HTML markup and provide a wicket:id. If you know Wicket, this is nothing new.

<div wicket:id="myForm"/>

Add the Form to the Wicket Page:

Wicked Forms provides the FormPanel class that can be added to the Java class of a Wicket page like this (again, if you know Wicket, this should be familiar):

add(new FormPanel("myForm", myForm));

That's it.

Bindings

Forms are useless if you cannot process the user's input. Thus, each form element can be bound to an arbitrary java object that stores the user's input. This concept is very similar to (and inspired by) Wicket's model concept, so if you know Wicket you will feel at home.

The Binding interface

The Binding interface is a simple interface that allows to get and set a value of a certain type. Very similar to Wicket's IModel interface. All form elements have a constructor that expects a Binding implementation as parameter. That Binding connects the user input to the Java object that backs the Binding.

SimpleBinding

The simplest possible implementation of the Binding interface is SimpleBinding. It is used implicitly if no Binding implementation is passed to a form element. You won't really need to use it.

PropertyBinding

A PropertyBinding binds the user input of a form element to an instance variable of a Java object. The following code binds the input of a text field to the "name" property of a Person object:

Person person = new Person();
person.setName("Tom");				
Binding binding = new PropertyBinding(person, "name");				
StringTextFieldModel textField = new StringTextFieldModel("Label", binding)));

Custom Bindings

You are free to create your own implementations of the Binding interface that suite your project's needs. If you come up with implementations that may be of benefit to others, let us know and we will incorporate them into Wicket Forms :).
